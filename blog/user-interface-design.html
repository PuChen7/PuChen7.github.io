General Introduction to GUI
Important about GUI design
  • Not just think as user, you need to imagine what your users are like. How would they use the feature? etc.
  • Walk the user through the GUI. Put yourself in user's position
Good UI should be:
  • Attractive
  • Simple to use
  • Responsive in short time
  • Clear to understand
  • Consistent on all interfacing screens

User Interface Design Activities
  • GUI Requirement Gathering: Taken from users.
  • User Analysis: Who is going to use the software? Technical savvy? Novice user?
  • Task Analysis: what tasks need to be done?
  • GUI Design & implementation: design and implement the GUI
  • Test: involvement of users and release of beta version are few of them. Testing may include usability, compatibility, user acceptance etc.
User Interface Golden rules
  • Consistency: e.g. a certain style of button should always do the same thing.
  • Informative feedback: The system should always keep users informed about what is going on. e.g. installation status
  • Permit easy reversal of actions
  • Error handling
  • Using short-cuts

HTML Concepts
What is HTML: Hyper Text Markup Language.
DOCTYPE: an instruction to the web browser about what version of HTML the page is written in. It should be the very first thing in an HTML document.

Differences Between SVG and Canvas

  • SVG is a language for describing 2D graphics in XML.
  • Canvas draws 2D graphics, on the fly (with a JavaScript).
  • SVG is XML based, which means that every element is available within the SVG DOM. You can attach JavaScript event handlers for an element.
  • In SVG, each drawn shape is remembered as an object. If attributes of an SVG object are changed, the browser can automatically re-render the shape.
  • Canvas is rendered pixel by pixel. In canvas, once the graphic is drawn, it is forgotten by the browser. If its position should be changed, the entire scene needs to be redrawn, including any objects that might have been covered by the graphic.

CSS Concepts
Cascading Order
What style will be used when there is more than one style specified for an HTML element?
Generally, all the styles will "cascade" into a new "virtual" style sheet by the following rules, where number one has the highest priority:
  • Inline style (inside an HTML element)
  • External and internal style sheets (in the head section)
  • Browser default
So, an inline style (inside a specific HTML element) has the highest priority, which means that it will override a style defined inside the <head> tag, or in an external style sheet, or a browser default value.

CSS Selectors
Selectors are what allows you to target specific HTML elements and apply style to them.

ID vs Class
  • ID is unique
    • Each element can have only one ID
    • Each page can have only one element with that ID
    • Use a class when you want to consistently style multiple elements throughout the page/site.
  • Classes are NOT unique
    • You can use the same class on multiple elements.
    • You can use multiple classes on the same element.
    • Use the ID when you have a single element on the page that will take the style.

absolute, relative, fixed and static position
  • static: this is the default for web page. The element normally displayed.
  • relative: relative to itself. If you set position: relative; without any other attributes, it is the same as static. But if you do give it some other positioning attribute, say, top: 10px;, it will shift its position 10 pixels down from where it would normally be.
  • absolute: place any page element exactly where you want it. An element with this type of positioning is not affected by other elements and it doesn't affect other elements.
  • fixed: fixed position element is positioned relative to the viewport. A fixed positioned element will stay right where it is when the page is scrolled.

Inline, block, and inline-block
inline: respect left and right, not top and bottom. Cannot have width and height.
block: respect left, right, top, bottom. Force line break after the block element.
inline-block: allow other to sit at right and left. Respect top, bottom margins and padding. Respect height and width.

Pseudo class
pseudo class tells you specific state of an element. allow to style element dynamically.
Example: :hover, :visited, and :nth-child.
